# Handoff: 현장출근기록 (인테리어 현장 출근 관리)

> 인테리어 공사 현장의 **일별 출근 현황**을 기록·조회하는 모바일 우선 웹 앱.
> 현장 관리기사 여러 명이 함께 사용. 목표 스택: **Next.js + TypeScript + Supabase + Vercel**.

---

## ⚠️ 먼저 읽어주세요 — 이 번들의 성격

이 폴더의 HTML/JSX 파일은 **디자인 레퍼런스(프로토타입)** 입니다. 의도한 **모양과 동작**을 보여주는
참고물이지, 그대로 복사해 배포할 프로덕션 코드가 아닙니다.

할 일은 이 디자인을 **타깃 코드베이스(Next.js + TypeScript)** 의 패턴·라이브러리로 **다시 구현**하는 것입니다.
프로토타입은 React + 인라인 스타일 + 단일 HTML로 빠르게 만든 것이며, 실제 프로젝트에서는:
- **Next.js App Router** + 서버/클라이언트 컴포넌트 분리
- 스타일링: **Tailwind CSS** 또는 CSS Modules (아래 디자인 토큰을 그대로 옮기면 됨)
- 데이터: **Supabase** (Postgres + Auth + RLS) — 아래 스키마 제안 참고
- 인증: Supabase Auth (이메일/비번 + **카카오 OAuth**)

로 재구성하는 것을 권장합니다.

## 화면 구성 (총 10개 뷰 + 로그인)

내비게이션 구조:
- **PC (≥880px)**: 좌측 사이드바 (대시보드 · 현장 · 통합 달력 · 월별 통계 · 노무비 · 공종·업체) + 하단 프로필(→설정)
- **모바일 (<880px)**: 상단 바 + 하단 탭바 5개 (홈 · 현장 · 달력 · 통계 · 더보기). '더보기'에 노무비·공종·업체·설정 모음

브레이크포인트: **880px** (이상 = wide/PC, 미만 = 모바일)

### 1. 로그인 (`LoginScreen`)
- **목적**: 자체 이메일/비번 로그인 + 회원가입, 카카오 간편 로그인/가입
- **레이아웃**: 세로 중앙 정렬, max-width 380px. 상단 브랜드 로고(64px 둥근 사각, 블루) → 제목 "현장출근기록" → 카드
- **컴포넌트**:
  - 세그먼트 컨트롤(로그인/회원가입 전환) — 카드 상단 full-width
  - 이메일·비밀번호 입력 필드 (좌측 아이콘)
  - 회원가입 모드에선 "이름" 필드 추가
  - 기본 로그인 버튼(블루, full, lg) + "또는" 구분선 + **카카오 버튼**(`#FEE500` 배경, `#191600` 텍스트, 카카오 아이콘)
- **동작**: 로그인/카카오 버튼 → 인증 성공 시 대시보드로

### 2. 대시보드 (`DashboardScreen`)
- **목적**: 오늘/최근 7일 출근 요약
- **레이아웃**: 인사말 헤더 → Hero 그리드(PC 2열, 모바일 1열) → 현장별 오늘 출근 카드 그리드(PC 2열)
- **컴포넌트**:
  - **오늘 전체 출근** 카드: 블루 그라데이션 배경(`135deg, #2563EB→#1D4ED8`), 흰 텍스트, 큰 숫자(46px/800) + 진행현장 수 배지 + 하단 구분선 위 최근 7일 man·day/공종 수
  - **최근 7일 추이** 카드: 미니 막대 그래프(오늘은 진한 블루, 나머지는 `--blue-100`), 높이 96px
  - **현장별 오늘 출근** 카드들: 현장명 + 주소 + 총 인원(또는 "미입력" 앰버 배지) + 공종별 인원 칩
- **동작**: 현장 카드 클릭 → 현장 상세

### 3. 현장 관리 (`SiteListScreen`)
- **목적**: 현장 목록 + 추가/수정/삭제, 검색
- **레이아웃**: 헤더(제목 + PC는 "현장 추가" 버튼) → 검색 입력 → 카드 그리드(PC 2열)
- **컴포넌트**:
  - 검색 입력(돋보기 아이콘)
  - 현장 카드: 현장명 + 상태 배지(진행중=블루 / 마감임박=앰버 / 완료=슬레이트) + 주소 + 담당자 + 오늘 인원, 하단에 "출근 기록"·"수정" 버튼
  - 모바일: 우하단 **FAB**(58px, 블루, +아이콘, `bottom:92px`)
  - 추가/수정 **바텀시트**: 현장 이름·주소·담당 관리기사 필드, (수정 시)삭제 버튼
- **동작**: 카드/출근기록 → 상세, 수정 → 시트, FAB/추가 → 새 현장 시트

### 4. 현장 상세 (`SiteDetailScreen`)
- **목적**: 한 현장의 출근 입력·조회·일지. 4개 탭
- **레이아웃**: 뒤로가기 + 현장명/주소/상태 헤더 → **세그먼트 탭(입력·달력·표·일지)** → (입력/일지 탭은 가로 날짜 스트립) → 탭 콘텐츠
- **탭 콘텐츠**:
  - **입력** (`AttendanceInput`): 상단 요약 바(날짜·공종수·총인원) + 입력 UI(아래 3가지 변형 중 하나)
  - **달력** (`CalendarView`): 월간 히트맵 달력(인원 많을수록 진한 블루) + 우측 선택일 상세
  - **표** (`TableView`): 기간(7/14/30일) × 공종 매트릭스, 행/열 합계, CSV 내보내기
  - **일지** (`JournalView`): 현장 사진(드래그 업로드 슬롯) + 작업 일지 메모(textarea)
- **날짜 스트립** (`DateStrip`): 가로 스크롤, 오늘 기준 ±, 선택일 블루 강조, 요일 색(일=빨강/토=블루)

#### 출근 입력 — 3가지 변형 (Tweaks로 전환, 하나로 확정 필요)
- **A. 스테퍼 카드** (`InputStepper`, 기본): 공종별 카드 + 큰 −/+ 스테퍼 + 메모 토글. **현장 추천**
- **B. 숫자패드** (`InputPad`): 공종 타일 그리드 선택 → 큰 숫자패드(1~9,C,0,⌫)로 입력
- **C. 리스트 인라인** (`InputInline`): 한 줄에 공종·스테퍼·메모. 정보 밀도 높음, PC 친화
- 입력은 **즉시 자동 저장** (별도 저장 버튼 없음)

### 5. 통합 달력 (`UnifiedCalendarScreen`)
- **목적**: 여러 현장을 한 달력에 겹쳐 보기
- **레이아웃**: 현장 토글 칩(켜고 끄기, 현장별 색상 점) → 달력(PC 1.5:1로 우측 선택일 패널)
- **컴포넌트**: 각 날짜 셀에 현장별 색상 점(최대 6개) + 합계 인원. 우측 패널: 선택일 현장별 인원(클릭 시 해당 현장 상세로)
- **현장 색상 팔레트**: `#2563EB, #0EA5E9, #8B5CF6, #F59E0B, #14B8A6, #EC4899, #6366F1, #EF4444` (현장 인덱스 순)

### 6. 월별 통계 (`StatsScreen`)
- **목적**: man-day 집계·분석
- **레이아웃**: 월 선택기 + 현장 필터 칩 → KPI 4개 → 일별 추이 막대 → 공종별/현장별 집계 바(PC 2열)
- **KPI**: 총 man-day(블루 강조 카드) · 작업일수 · 일 평균 · 활성 공종

### 7. 노무비 계산 (`PayrollScreen`)
- **목적**: 공종별 **일당 × man-day = 노무비** 집계
- **레이아웃**: 월 선택기 + 현장 필터 칩 → 총 노무비 Hero(블루 그라데) → 공종별 노무비 표 → 비중 바 차트
- **표 컬럼**: 공종·업체 / 일당 / man·day / 노무비. 행/합계, 정산서 내보내기 버튼
- **계산**: `노무비 = man_day × trade.rate`. 통화는 `toLocaleString('ko-KR')` (원)

### 8. 공종 · 업체 관리 (`TradeManageScreen`)
- **목적**: 공종(=협력사) 카테고리 추가/수정. 출근 입력에 쓰이는 항목
- **레이아웃**: 헤더(+버튼/FAB) → 카드 그리드(PC 3열)
- **카드**: 색상 점 + 공종명 + 업체명 + 일당 + 누적 man·day. **클릭 → 업체 상세**, 우측 수정 버튼
- **수정 시트**: 공종명 · 업체명 · 담당자 · 연락처 · 일당(숫자) · 컬러 팔레트(12색)

### 9. 업체(협력사) 상세 (`VendorDetailScreen`)
- **목적**: 업체별 연락처·투입 이력
- **레이아웃**: 뒤로가기 → 헤더 카드(업체명·공종 배지·담당자·연락처·전화/수정 버튼) → KPI 4개(일당·누적 man·day·누적 노무비·투입 현장 수) → 투입 이력 리스트(날짜·현장·메모·인원, 최근 30건)

### 10. 내 정보 · 설정 (`SettingsScreen`)
- **목적**: 프로필·계정·알림 관리
- **레이아웃**: max-width 720px 중앙. 프로필 카드(아바타·이름·역할/회사·편집 + 담당현장/누적입력/가입 통계) → 계정 섹션(이메일·휴대폰·비번변경·카카오 연결) → 알림 섹션(토글 3개) → 로그아웃
- **토글**: 미입력 현장 알림 · 주간 요약 리포트 · 업체 투입 변동 알림

## 인터랙션 & 동작
- **자동 저장**: 출근 입력·일지 메모는 변경 즉시 저장 (저장 버튼 없음). "자동 저장됩니다" 안내 표기.
- **네비**: PC 사이드바 / 모바일 하단탭. 880px에서 전환 (resize 리스너).
- **바텀시트**: 백드롭 클릭으로 닫힘, `slideUp .26s cubic-bezier(.16,1,.3,1)` 등장.
- **달력 히트맵**: 인원/최댓값 비율로 블루 투명도 `0.12 + ratio*0.7`. 오늘 표시 점.
- **막대/바 차트**: `width/height transition .5s`로 등장.
- **반응형**: 표/칩 줄은 가로 스크롤 허용. 그리드는 PC 다열 → 모바일 1열.

## 상태 관리 (프로토타입 기준 → 실제는 Supabase 연동)
- `sites[]`, `trades[]`(공종+업체+일당), `records{ "siteId|YYYY-MM-DD": { tradeId: {count, memo} } }`, `journal{ "siteId|date": {memo, photos} }`
- `route`(현재 화면 + siteId/tradeId), `authed`, 입력 변형(`inputVariant`), 강조색(`accent`)
- 실제 구현: 위 데이터는 Supabase 테이블로, 화면 상태는 URL 라우팅(App Router)으로.

## Assets
- **아이콘**: 모두 인라인 SVG(stroke 라인 아이콘, 24x24 viewBox). 실제 구현 시 `lucide-react` 등으로 대체 권장 (현재 아이콘은 lucide와 거의 1:1 매칭: dashboard, building, calendar, table, wrench, bar-chart, user, plus, minus, check, x, chevron, edit, trash, search, menu, log-out, map-pin, bell, zap, download 등).
- **폰트**: Pretendard (CDN 또는 `pretendard` npm 패키지).
- **카카오 로고**: 인라인 SVG (로그인 버튼).
- **현장 사진**: 일지 탭의 드래그 업로드 슬롯 — 실제로는 Supabase Storage 버킷에 업로드.
- 이미지/사진 placeholder만 사용, 실제 사진 없음.
